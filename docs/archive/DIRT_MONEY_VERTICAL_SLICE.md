# DIRT MONEY — Vertical Slice (Scope Contract)
**Nothing enters the slice that is not in the sprint table below. New ideas go to DIRT_MONEY_CRITIQUE.md.**

## Target
4–6 hours playable · 30 in-game days · 3 backgrounds · Ash Creek County · corn/soy/hay · chickens (feed/collect/sell).

## In scope (systems)
Movement (node travel) · Calendar · Weather (7 states) · Economy/market · Inventory · Contracts (5 types, via conversation) · Equipment (6 subsystems) · Repairs · Events (10–12) · Travel · Skill progression (3 mini trees, ~8 perks each) · Save/load · Market fluctuations · Dialogue/skill-check engine + ReputationLedger · Talking portraits + gibberish audio.

## Explicitly cut from slice
- Full auction system → Mechanic-only **scripted events**; full system Phase 3
- Construction → Phase 3
- NPC schedules/gifting → single reputation number per NPC
- Chicken breeding → feed/collect/sell only
- Six-tree perk system → one mini tree per background
- Seasons → single window; Phase 3
- Ranch Daughter / Market Operator backgrounds → Phase 3 (gated)

## Phase 1 sprint table (weeks 3–16)

| Sprint | Weeks | Deliverable |
|---|---|---|
| 1 | 3–4 | Character select, save system skeleton, calendar/day loop, time-of-day schedule |
| 2 | 5–6 | **Dialogue/skill-check engine** (data-driven trees, visible-odds checks, ReputationLedger) + talking-portrait stub (placeholder busts, jaw-flap, gibberish audio component) — built early: core system everything plugs into |
| 3 | 7–8 | Travel node map (~10 locations), fuel/time costs, inventory |
| 4 | 9–10 | Crop system (corn/soy/hay) as one-tap orders + ambient animation + zoom view; weather (7 states) |
| 5 | 11 | Economy + market fluctuation, elevator selling, loans/debt tick (terms via dialogue checks) |
| 6 | 12 | Contracts through NPC conversations (5 types, rep-gated tiers); chickens |
| 7 | 13 | Equipment condition + repair (6 subsystems; Mechanic full detail; repair negotiation dialogue) |
| 8 | 14 | Background exclusives: Read The Land, Farm Dashboard, Salvage events; 3 mini perk trees (perks unlock dialogue options) |
| 9 | 15 | Random events (10–12) as interrupts; end-of-run report card + reputation summary; run comparison; endings |
| 10 | 16 | Balance pass vs Phase 0 spreadsheet; internal 30-day playthrough per background verifying ripples change options |

**Playable-every-sprint rule:** each sprint ends with a build the director can play.

## Exit criteria (Phase 1)
Director completes Day 30 as all three backgrounds, and each run **felt** different — not just numerically different.

## Kill/pivot gate (Phase 2)
If <20% of playtesters voluntarily start a second background: **fix backgrounds before adding any content.**
